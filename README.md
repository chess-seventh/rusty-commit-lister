## Overview

`rusty-commit-lister` is a companion tool for [`rusty-commit-saver`](https://github.com/franci/rusty-commit-saver).

`rusty-commit-saver` works as a Git post-commit hook and saves every commit
into your Obsidian Diary daily notes as a Markdown table.
`rusty-commit-lister` reads those daily notes, parses the commit tables, and
presents everything in a clean and interactive terminal interface powered by
[`ratatui`](https://github.com/ratatui-org/ratatui).

---

## How It Works

```text
Obsidian Diary (Markdown files)
        │
        │  daily notes at path:
        │  ~/Documents/Wiki/📅 Diaries/0. Journal/YYYY/MM-MMM/YYYY-MM-DD.md
        │
        ▼
┌──────────────────────┐
│  rusty-commit-lister │  ← reads & parses commit tables from daily notes
└──────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────────────┐
│  ratatui TUI                                                 │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ date & hour  │ last dir  │ repo name  │ commit msg   │   │
│  └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

1. On startup, the app reads `~/.config/rusty-commit-lister/config.toml`.
2. It scans the Obsidian Diary directory for daily notes matching your
   configured date format.
3. For each daily note, it locates and parses the commit Markdown table
   inserted by `rusty-commit-saver`.
4. All parsed commits are loaded into an interactive `ratatui` table for
   browsing.

## Features

- Parses commit tables from Obsidian Diary daily notes generated by `rusty-commit-saver`
- Reads configuration from `~/.config/rusty-commit-lister/config.toml`
- Interactive terminal UI powered by `ratatui`
- Displays commits with:
  - Commit date and time
  - Last directory segment of the commit path
  - Repository name (extracted from the repository URL)
  - Commit message
- Navigable table with keyboard controls
- Supports multi-day browsing across your diary structure
- Lightweight and fast — pure Rust, no runtime dependencies

## Configuration

The configuration file lives at:

```text
~/.config/rusty-commit-lister/config.toml
```

If the directory does not exist, create it:

```bash
mkdir -p ~/.config/rusty-commit-lister
```

### Full config reference

```toml
# ~/.config/rusty-commit-lister/config.toml

# Root path of your Obsidian Diary
obsidian_diary_root = "/Users/franci/Documents/Wiki/📅 Diaries/0. Journal"

# Date format used to build daily note paths
# Example result: 2026/05-May/2026-05-16
daily_note_format = "%Y/%m-%B/%Y-%m-%d"

# File extension for daily notes
daily_note_extension = "md"

# Time display format in the TUI
time_format = "%H:%M"

# How many days back to scan by default (0 = today only)
scan_days_back = 7

# Optional: filter commits by repository name (leave empty to show all)
# repo_filter = ["my-repo", "another-repo"]
```

### Config field reference

| Field                  | Type     | Required | Default          | Description                                      |
|------------------------|----------|----------|------------------|--------------------------------------------------|
| `obsidian_diary_root`  | String   | Yes      | —                | Absolute path to your Obsidian Diary root        |
| `daily_note_format`    | String   | No       | `%Y/%m-%B/%Y-%m-%d` | strftime-like format for daily note paths    |
| `daily_note_extension` | String   | No       | `md`             | File extension for daily notes                   |
| `time_format`          | String   | No       | `%H:%M`          | Display format for commit time in the TUI        |
| `scan_days_back`       | Integer  | No       | `7`              | Number of days back to scan for commits          |
| `repo_filter`          | [String] | No       | `[]` (all)       | Optional list of repository names to include     |

## Usage

Launch the TUI:

```bash
rusty-commit-lister
```

### Keyboard controls

| Key           | Action                       |
|---------------|------------------------------|
| `↑` / `k`     | Move selection up            |
| `↓` / `j`     | Move selection down          |
| `PgUp`        | Scroll up one page           |
| `PgDn`        | Scroll down one page         |
| `Enter`       | Open commit details          |
| `r`           | Refresh / reload daily notes |
| `q` / `Esc`   | Quit                         |

### Optional flags

```bash
# Use a custom config file
rusty-commit-lister --config /path/to/config.toml

# Scan a specific daily note file directly
rusty-commit-lister --daily-note /path/to/2026-05-16.md

# Scan only today's note
rusty-commit-lister --today

# Scan a specific date
rusty-commit-lister --date 2026-05-14
```

## Output Format

Each row in the TUI displays:

```text
┌──────────────────────┬──────────────┬──────────────────┬──────────────────────────────────┐
│ Date & Time          │ Last Dir     │ Repository       │ Commit Message                   │
├──────────────────────┼──────────────┼──────────────────┼──────────────────────────────────┤
│ 2026-05-16  14:32    │ src          │ rusty-commit-... │ Fix OAuth token refresh          │
│ 2026-05-16  11:15    │ tests        │ my-api           │ Add integration test for /users  │
│ 2026-05-15  09:04    │ docs         │ obsidian-tools   │ Update README                    │
└──────────────────────┴──────────────┴──────────────────┴──────────────────────────────────┘
```

- **Date & Time** — extracted from the `TIME` field of the daily note table, combined with the note's date
- **Last Dir** — last segment of the `FOLDER` path (e.g., `src` from `/Users/franci/projects/my-repo/src`)
- **Repository** — repository name parsed from the `REPOSITORY URL` column
- **Commit Message** — raw commit message from the `COMMIT MESSAGE` column

## Daily Note Format

`rusty-commit-lister` expects daily notes to contain a Markdown table in the following format,
as generated by `rusty-commit-saver`:

```markdown
## Commits

| FOLDER                                      | TIME     | COMMIT MESSAGE              | REPOSITORY URL                                    |
| ------------------------------------------- | -------- | --------------------------- | ------------------------------------------------- |
| /Users/franci/projects/my-repo/src          | 14:32    | Fix OAuth token refresh     | https://github.com/franci/my-repo                 |
| /Users/franci/projects/obsidian-tools/docs  | 11:15    | Update README               | https://github.com/franci/obsidian-tools          |
```

The daily note files are expected at paths following this structure:

```text
{obsidian_diary_root}/{YYYY}/{MM-Month}/{YYYY-MM-DD}.md

Example:
/Users/franci/Documents/Wiki/📅 Diaries/0. Journal/2026/05-May/2026-05-16.md
```
